ami ( 0.2.25p )

Arguments

  • context

    --context characterExtent [characterExtent2]{1,2}
    Default: 99Methods: parse: parseContext

    b: c: java.lang.Integer

    Description

    CONTEXT: The extent (in characters) of the text or other content immediately surrounding the located word/phrase/object. By default this is either +- 100 characters (Publisher limit) or to the end of the object (paragraph). a single "characterExtent" field gives +- characterExtent (post == pre), while "characterExtent "characterExtent2" gives the preceeding and following text. Note: --context must come before any arguments which may depend on its value (e.g. regex)

  • param

    --param string [string]{1,*}
    Methods: parse: parseParam

    b: -p c:

    Description

    PARAM A simple method (primarily for demo and learning) that passes a number of strings to the application. The strings are parsed by "parseParam()" which should be overridden in the application. The application consists of subclasses of SimpleVisitor and SimpleSearcher

  • update

    --update (overwrite|merge|none){1,1}
    Constraints: true Default: overwriteMethods: parse: parseUpdate

    b: -u c:

    Description

    UPDATE If an existing results file exists choose the option for updating: "merge" is not yet supported and will throw an Exception "none" will throw an exception "overwrite" will overwrite

  • lookup

    --lookup site [site...]{1,*}
    Constraints: trueMethods: parse: parseLookup final: finalLookup

    Description

    Looks up results in online sites. This can be overridden for specialist plugins. may be called at ResultElement creation or might be deferred to final.

  • test

    --test1 string [string ...]{1,*}
    Constraints: trueMethods: parse: parseTest

    b: -t1 c:

    Description

    A dummy parameter to test conformance

  • stopwords

    --w.stopwords location [location...]{0,*}
    Methods: parse: parseStopwords

    Description

    STOPWORDS: select sets of stopwords. Locations are given by resource locations.

  • case

    --w.case options{1,*}
    Methods: parse: parseCase

    b: c: java.lang.String

    Description

    CASE respects or ignores case: * default is to respect case * ignore = ignore case (everything is internally lowercased) * respectCamel = don't lowercase anything with midWord uppercase (URL, eBook,)

  • stemming

    --w.stem {1,1}
    Default: falseMethods: parse: parseStem

    b: c: java.lang.Boolean

    Description

    STEM Stemming removes computable suffixes from words (e.g. walks, walked => walk) Uses the Porter Stemming algorithm in Lucene.
